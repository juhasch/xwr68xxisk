# Configuration (.cfg) File Format

Each line in the .cfg file describes a command with parameters. The various commands and their arguments are described in the table below (arguments are in sequence).


## List of commands

Taken from "mmwave_sdk_<ver>\packages\ti\control\mmwavelink\docs\doxygen\html\index.html". Additionally, users can refer to the chirp diagram below to understand the chirp and profile related parameters or the appnote <http://www.ti.com/litv/pdf/swra553>

![](_static/chirp.jpeg)

Figure 2: Chirp Diagram

| Configuration Command | Command Details | Command Parameters | Usage in mmW demo |
|---------------------|-----------------|-------------------|------------------|
| dfeDataOutputMode | The values in this command should not change between sensorStop and sensorStart. Reboot the board to try config with different set of values in this command. This is a mandatory command. | **modetype**:<br>• 1 - frame based chirps<br>• 2 - continuous chirping<br>• 3 - advanced frame config | Only option 1 and 3 are supported |
| channelCfg | Channel config message to RadarSS. See mmwavelink doxgen for details. The values in this command should not change between sensorStop and sensorStart. This is a mandatory command. | **rxchannelen**:<br>• Receive antenna mask e.g for 4 antennas, it is 0x1111b = 15<br><br>**txchannelen**:<br>• Transmit antenna mask<br><br>**cascading**:<br>• SoC cascading, not applicable, set to 0 | • 4 antennas supported<br>• For antenna configuration:<br>&nbsp;&nbsp;- IWR6843 ISK: Use 0x5 for 2 azimuth antennas (tx1, tx3), 0x7 for both azimuth and elevation (tx1, tx2, tx3)<br>&nbsp;&nbsp;- xWR1642BOOST: Use 0x3 for 2 azimuth antennas (tx1, tx2)<br>• Cascading: n/a |
| adcCfg | ADC config message to RadarSS. See mmwavelink doxgen for details. The values in this command should not change between sensorStop and sensorStart. This is a mandatory command. | **numadcbits**:<br>• 0 - 12-bits<br>• 1 - 14-bits<br>• 2 - 16-bits<br><br>**adcoutputfmt**:<br>• 0 - real<br>• 1 - complex 1x (filtered)<br>• 2 - complex 2x (unfiltered) | • Only 16-bit is supported<br>• Only complex modes are supported |
| adcbufCfg | adcBuf hardware config. The values in this command can be changed between sensorStop and sensorStart. This is a mandatory command. | **subframeidx**:<br>• subframe Index<br><br>**adcoutputfmt**:<br>• 0 - Complex<br>• 1 - Real<br><br>**sampleswap**:<br>• 0 - I in LSB, Q in MSB<br>• 1 - Q in LSB, I in MSB<br><br>**chaninterleave**:<br>• 0 - interleaved (XWR14xx only)<br>• 1 - non-interleaved<br><br>**chirpthreshold**:<br>• Threshold for buffer switch (0-8) | • Legacy mode: subframeidx = -1<br>• Advanced frame: use subframe number or -1<br>• Only complex modes supported<br>• Only option 1 for sampleswap<br>• Only option 1 for chaninterleave<br>• xwr16xx: 0-8 supported (1 for LVDS)<br>• Other platforms: only value 1 |
| profileCfg | Profile config message to RadarSS and datapath. See mmwavelink doxgen for details. The values in this command can be changed between sensorStop and sensorStart. This is a mandatory command. | **profileid**:<br>• Profile Identifier<br><br>**startfreq**:<br>• Frequency Start in GHz<br><br>**idletime**:<br>• Idle Time in μs<br><br>**adcstarttime**:<br>• ADC Valid Start Time in μs<br><br>**rampendtime**:<br>• Ramp End Time in μs<br><br>**txoutpower**:<br>• TX power back-off code<br><br>**txphaseshifter**:<br>• TX phase shifter<br><br>**freqslopeconst**:<br>• Freq slope (MHz/μs)<br><br>**txstarttime**:<br>• TX Start Time (μs)<br><br>**numadcsamples**:<br>• ADC sample count<br><br>**digoutsamplerate**:<br>• ADC freq (ksps)<br><br>**hpfcornerfreq1/2**:<br>• HPF settings<br><br>**rxgain**:<br>• RX gain (dB) | • Legacy: One profile per config<br>• Advanced: One profile per subframe<br>• All timing per datasheet<br>• txoutpower=0, txphaseshifter=0<br>• freqslopeconst > 0<br>• HPF1: 175/235/350/700 KHz<br>• HPF2: 350/700/1400/2800 KHz |
| calibDcRangeSig | DC range calibration configuration | **subFrameIdx**:<br>• Subframe index<br><br>**enabled**:<br>• Enable/disable calibration<br><br>**negativeBinIdx**:<br>• Negative bin index<br><br>**positiveBinIdx**:<br>• Positive bin index<br><br>**numAvgFrames**:<br>• Number of frames to average | Example:<br>`calibDcRangeSig -1 0 -5 8 16` |
| clutterRemoval | Clutter removal configuration | **subFrameIdx**:<br>• Subframe index<br><br>**enabled**:<br>• Enable/disable clutter removal | Example:<br>`clutterRemoval -1 0` |
| compRangeBiasAndRxChanPhase | Range bias and RX channel phase compensation | **rangeBias**:<br>• Range bias value<br><br>**Re/Im[XX]**:<br>• Real and imaginary components for each channel | Complex calibration parameters for RX channels |
| aoaFovCfg | Angle of Arrival Field of View configuration | **subFrameIdx**:<br>• Subframe index<br><br>**minAzimuthDeg**:<br>• Minimum azimuth in degrees<br><br>**maxAzimuthDeg**:<br>• Maximum azimuth in degrees<br><br>**minElevationDeg**:<br>• Minimum elevation in degrees<br><br>**maxElevationDeg**:<br>• Maximum elevation in degrees | Sets the field of view limits for angle of arrival processing |
| cfarCfg | CFAR detection configuration | **subFrameIdx**:<br>• Subframe index<br><br>**procDirection**:<br>• 0: range, 1: doppler<br><br>**averageMode**:<br>• Averaging mode<br><br>**winLen**:<br>• Window length<br><br>**guardLen**:<br>• Guard length<br><br>**noiseDiv**:<br>• Noise divider<br><br>**cyclicMode**:<br>• Cyclic mode<br><br>**thresholdScale**:<br>• Threshold scale<br><br>**peakGroupingEn**:<br>• Peak grouping enable | Example:<br>`cfarCfg -1 0 2 8 4 3 0 15 1` |
| multiObjBeamForming | Multi-object beamforming configuration | **subFrameIdx**:<br>• Subframe index<br><br>**enabled**:<br>• Enable/disable<br><br>**threshold**:<br>• Detection threshold | Example:<br>`multiObjBeamForming -1 1 0.5` |
| guiMonitor | GUI monitoring configuration | **detectedObjects**:<br>• 0: None<br>• 1: Objects + side info<br>• 2: Objects only<br><br>**logMagRange**:<br>• Log magnitude range<br><br>**noiseProfile**:<br>• Noise floor profile<br><br>**rangeAzimuthHeatMap**:<br>• Range-azimuth heat map<br><br>**rangeDopplerHeatMap**:<br>• Range-Doppler heat map<br><br>**statsInfo**:<br>• Statistics information | Example:<br>`guiMonitor -1 1 1 0 0 0 1` |
| analogMonitor | Analog monitoring configuration | **rxSaturation**:<br>• RX saturation monitoring<br><br>**sigImgBand**:<br>• Signal image band monitoring | Example:<br>`analogMonitor 0 0` |
| lvdsStreamCfg | LVDS streaming configuration | **subFrameIdx**:<br>• Subframe index<br><br>**enableHeader**:<br>• Enable header<br><br>**dataFmt**:<br>• Data format<br><br>**enableSW**:<br>• Enable software | Controls LVDS data streaming |
